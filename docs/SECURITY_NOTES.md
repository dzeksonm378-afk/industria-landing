# Security Notes

## Ограничения Stage 0/1/2/3/4/4.5/4.6/4.8

На текущих frontend-этапах проект остается статическим frontend-demo:

- backend не добавляется;
- API routes не добавляются;
- база данных не добавляется;
- авторизация не добавляется;
- платежи не добавляются;
- CRM не добавляется;
- внешние скрипты не добавляются;
- секреты не хранятся в коде;
- `eval` не используется;
- `dangerouslySetInnerHTML` не используется.

## Формы и контактные CTA

На Stage 4.5-4.8 публичные формы убраны с главной страницы. Расчет по фото работает как frontend-only контактный сценарий:

- нет отправки данных;
- нет сохранения персональных данных;
- нет backend, API routes, CRM или email-интеграции;
- ссылки WhatsApp, Telegram, Instagram и телефона пока являются placeholder-значениями в `src/data/siteContent.ts`;
- перед запуском placeholder-ссылки нужно заменить на реальные контакты.

Перед запуском реальных форм нужно отдельно спроектировать:

- согласие на обработку персональных данных;
- политику конфиденциальности;
- валидацию на клиенте и сервере;
- защиту от spam/abuse;
- rate limiting на backend;
- безопасное хранение и передачу заявок;
- логирование без лишних персональных данных.

## Контент и placeholder-данные

Demo-кейсы и placeholder-цены не должны выдаваться за подтвержденные реальные данные. Portfolio-блок использует клиентское demo-описание и должен быть заменен на реальные кейсы после передачи материалов заказчиком.

Редактируемая цифра `500+ объектов` хранится в `src/data/siteContent.ts`. Если заказчик не подтвердит точную цифру, нужно заменить ее на мягкую формулировку.

## Accessibility and UI safety

На Stage 1-4.8 заложены:

- `aria-label` у важных CTA и визуальных placeholder-блоков;
- alt/aria-описания для мест будущих изображений;
- focus states;
- контрастная темная тема;
- отсутствие внешних трекеров и скриптов;
- логотип "ИнДустрия" используется из локального `public/brand/industria-logo.jpg`.

Stage 4.8 изменяет только frontend-брендинг: имя, слоган, локальный логотип, публичные тексты и metadata. Backend, API routes, внешние отправки, новые зависимости и реальные формы не добавлялись.

## Что проверить перед production

- реальные формы и согласия;
- privacy policy;
- CSP/security headers;
- robots/sitemap;
- analytics consent, если будет аналитика;
- безопасность зависимостей;
- отсутствие секретов в репозитории;
- корректность всех claims: опыт, объекты, цены, сроки.
